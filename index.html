BBPLC Documentation

What is BBPLC?

BBPLC (Basic syntax Based Programming Language Compiler) is a low-level system programming language inspired by the philosophy of HLA (High-Level Assembler), the simplicity of BASIC syntax, and the hardware-oriented design of PL/M. It aims to provide structured low-level programming while remaining compact, transparent, and close to the machine.

The primary goal of BBPLC is to generate extremely compact and highly efficient native binaries by leveraging FASM as its backend assembler. The language focuses on minimal runtime overhead, predictable memory behavior, and direct control over execution flow.

BBPLC is not intended to compete with mature system languages such as Zig, Nim, or V. Instead, it is designed as a focused, lightweight tool for building small, fast, and architecture-aware native programs.

In its initial iterations, BBPLC targets x86 architecture (16-bit, 32-bit, and 64-bit) for Linux, Windows, and BIOS real-mode environments. The compilation model relies on generating FASM-compatible assembly as an intermediate step, allowing the production of native executables, object files, and dynamic libraries.

Why BBPLC exists

BBPLC exists as an experimental project driven by the desire to explore direct translation of high-level source statements into assembly instructions. The goal was to investigate how closely a structured language can map to machine-level behavior without introducing heavy abstractions or runtime overhead.

The language combines the ambition to remain close to assembly in expressive power while following a conceptual direction similar to PL/M — serving as a logical, simplified iteration of that idea in a modern context.

Philosophy

BBPLC follows three core principles:

  • Fast — minimal overhead and direct execution.
  • Compact — smallest possible binary output.
  • Minimal — no unnecessary abstractions or hidden mechanisms.

Grammar

  • PRINT — outputs a value to the console.
  • PRTLN — just prints new line
  • READ — reads input from the user.
  • TOSTR — converts a number to a string.
  • TOINT — converts a string to a number.
  • IF — starts a conditional block.
  • THEN — marks the true branch of a condition.
  • ELSE — marks the false branch of a condition.
  • ENDIF — ends a conditional block.
  • LABEL — defines a label for jumps.
  • GOTO — performs an unconditional jump to a label.
  • DECLARE — declares and optionally initializes a variable.
  • RESERVE — reserves memory space without initialization.
  • MOV — moves a value into a variable.
  • ADD — adds two values.
  • MUL — multiplies two values.
  • DIV — divides one value by another.
  • SQR — calculates the square of a value.
  • POW — raises a value to a specified power.
  • PUSH — pushes a value onto the stack.
  • POP — pops a value from the stack.
  • MACRO — defines a reusable macro block.
  • ENDMACRO — ends a macro definition.
  • PROC — defines a procedure (function).
  • ENDPROC — ends a procedure definition.
